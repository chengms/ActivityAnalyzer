# Electron 缓存错误说明

## ⚠️ 关于缓存错误

应用启动时可能会看到以下错误信息：

```
ERROR:cache_util_win.cc(20)] Unable to move the cache: 拒绝访问。 (0x5)
ERROR:disk_cache.cc(208)] Unable to create cache
ERROR:gpu_disk_cache.cc(676)] Gpu Cache Creation failed: -2
```

## 📋 这些错误的影响

### ✅ 不影响功能
- **应用可以正常运行**
- **核心功能不受影响**
- **数据追踪正常工作**
- **界面正常显示**

### ⚠️ 可能的影响
- **性能可能略有下降**（缓存未启用）
- **某些浏览器功能可能受限**（如 WebSQL）

## 🔍 错误原因

1. **权限问题**：Electron 尝试在系统目录创建缓存，但可能没有权限
2. **路径问题**：缓存路径可能被占用或无法访问
3. **GPU 缓存**：GPU 加速缓存创建失败

## ✅ 解决方案

### 方案 1: 忽略错误（推荐）

这些错误通常可以安全忽略，应用功能不受影响。

### 方案 2: 设置用户数据目录

可以在代码中设置用户数据目录，确保有写入权限：

```typescript
// 在 app.whenReady() 之前
app.setPath('userData', path.join(app.getPath('appData'), 'activity-analyzer'));
```

### 方案 3: 禁用硬件加速（如果不需要）

如果不需要 GPU 加速，可以禁用：

```typescript
app.disableHardwareAcceleration();
```

### 方案 4: 以管理员权限运行

某些情况下，以管理员权限运行可以解决权限问题。

## 🎯 建议

**对于大多数用户：** 可以安全忽略这些错误，应用功能完全正常。

**如果错误频繁出现：**
1. 检查应用是否有写入权限
2. 尝试以管理员权限运行
3. 检查防病毒软件是否阻止了文件访问

## 📝 技术说明

这些错误来自 Chromium（Electron 的基础），是内部缓存系统的警告。它们不会：
- ❌ 阻止应用启动
- ❌ 影响数据存储
- ❌ 影响活动追踪
- ❌ 影响界面显示

只会可能影响：
- ⚠️ 网页缓存性能
- ⚠️ GPU 加速（如果启用）

---

**结论：** 这些错误可以安全忽略，应用功能完全正常。
